# JPD Ticket Template

This is the template structure the tool uses to generate the JPD summary output. Update section headings, descriptions, or guidance text here to keep the tool aligned with any changes to the official JPD template in Jira Product Discovery.

Each section heading is followed by the guidance shown to the submitter in the original JPD form. The tool uses these headings and descriptions to structure the generated markdown output.

---

## How was this need identified

e.g. Patient feedback / complaint, user research, policy driver, Regulatory / Compliance obligation.

---

## Is there a deadline or dependency driving this

Yes / No. If yes: provide information about the obligation type (GDPR / HIQA / EHDS / eHealth / Programme milestone / Other) and date or detail.

---

## Supporting evidence

Provide any evidence that supports the need for this feature. This helps the product team understand the strength of the case and speeds up discovery.

Examples:
- Link to user research or usability testing findings
- Volume of complaints or HSE Live contacts about this issue
- Clinical guideline, policy document or programme specification
- Data showing the scale of the problem (e.g. number of patients affected)
- Quotes or feedback from patients, carers or staff

If you do not have formal evidence, a brief description of where the need came from is still useful.
